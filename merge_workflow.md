```

    ↓
```

### 2.4 审批标准

**必须满足**：
- [ ] 代码风格符合规范

- [ ] 所有测试通过

- [ ] 无安全漏洞

**建议满足**：
- [ ] 代码有适当注释
- [ ] 遵循DRY原则
- [ ] 适当的错误处理
- [ ] 日志记录完整

## 六、特殊情况处理

### 6.1 紧急合并

**触发条件**：线上故障修复、安全漏洞修复

**流程**：
1. 技术总监口头通知用户
2. 用户口头授权
3. 技术总监执行合并
4. 事后补全审批文档

**审批时限**：口头授权即可执行

### 6.2 合并冲突

**处理流程**：
1. 识别冲突
2. 分析冲突原因
3. 制定解决方案
4. 技术总监审批方案
5. 执行解决
6. 重新提交合并申请

### 6.3 合并拒绝

**处理流程**：
1. 技术总监记录拒绝原因
2. 反馈给SubAgent
3. SubAgent修改后重新提交
4. 重新进入审批流程
