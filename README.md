## 🚀 AI开发工作流

### 核心流程（10个阶段）

```
需求分析 → 需求评审 → UI设计 → UI评审 → 开发设计 → 开发评审 → 开发自测 → 测试 → 部署 → 知识沉淀
```

### AI组织结构

```
                    ┌─────────────────────────────────────┐
                    │           主Agent (总指挥)            │
                    │  - 任务分解与分配                    │
                    │  - 进度监控与协调                    │
                    │  - 质量把控与决策                    │
                    └─────────────────┬───────────────────┘
                                      │
         ┌────────────────────────────┼────────────────────────────┐
         ▼                            ▼                            ▼
┌─────────────────┐      ┌─────────────────┐      ┌─────────────────┐
│   需求Agent    │      │   UI设计Agent   │      │  开发Agent     │
│ - 需求分析      │◄────►│ - 界面设计      │◄────►│ - 架构设计     │
│ - 需求建模      │      │ - 交互设计      │      │ - 代码实现     │
│ - 需求文档      │      │ - 设计评审      │      │ - 代码评审     │
└─────────────────┘      └─────────────────┘      └─────────────────┘
```

### 避免AI幻觉策略

- ✅ 多Agent交叉验证
- ✅ 事实核查机制
- ✅ 分阶段验证
- ✅ 人机协作确认

## 📋 目录结构规范

```
code/
├── 开发规范/              ← 全局开发规范（所有项目共享）
│   ├── AI开发工作流规范.md
│   ├── Git工作流.md
│   ├── 开发规范.md
│   ├── 需求管理规范.md
│   └── merge_workflow.md
│
├── 需求管理系统/          ← 项目代码（开发过程文档放这里）
│   ├── 00-需求池/
│   ├── 01-需求评审/
│   ├── 02-UI设计/
│   ├── 03-测试/
│   ├── 04-开发设计/
│   └── 05-工作流规范/
│
├── 原型演示系统/          ← 项目代码
└── 其他项目...            ← 项目代码
```

**重要规则**：
- ✅ 开发规范属于全局，放在 `code/开发规范/`
- ✅ 开发过程文档属于项目，放在各项目文件夹
- ❌ 禁止在 `code/` 根目录放置项目代码
- ❌ 禁止在开发规范目录放置项目代码
