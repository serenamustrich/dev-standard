# 项目规则

**版本**：v1.0
**更新**：2026-03-27

## 一、目录结构规则

### 1.1 code目录结构

```
code/
├── 开发规范/              ← 全局规范（所有项目共享）
├── 需求管理系统/          ← 项目代码
├── 原型演示系统/          ← 项目代码
└── 其他项目/              ← 项目代码
```

### 1.2 规则

- ✅ 开发规范属于全局，放在 `code/开发规范/`
- ✅ 开发过程文档属于项目，放在各项目文件夹
- ❌ 禁止在 `code/` 根目录放置项目代码
- ❌ 禁止在开发规范目录放置项目代码

## 二、Git分支规则

### 2.1 分支类型

| 分支 | 用途 | 保护规则 |
|------|------|----------|
| `main` | 生产环境代码 | 必须通过用户指令合并 |
| `test` | 测试环境代码 | 所有开发在此测试 |
| `sub/<Agent>/<任务>` | SubAgent开发分支 | 完成后合并到test |

### 2.2 开发流程

```
用户需求 → 主Agent分解 → SubAgent创建分支 → 开发测试 → 合并到test → 等待用户指令 → 合并到main
```

## 三、代码提交规则

### 3.1 提交信息格式

```
<类型>: <简短描述>

[详细描述]

关联需求: [需求编号]
```

### 3.2 类型

| 类型 | 说明 |
|------|------|
| feat | 新功能 |
| fix | 修复Bug |
| docs | 文档更新 |
| style | 代码格式 |
| refactor | 重构 |
| perf | 性能优化 |
| test | 测试 |
| chore | 构建/工具 |

### 3.3 示例

```bash
feat: 添加任务管理模块

- 创建Task entity
- 实现CRUD API
- 添加状态管理

关联需求: PRD-2026-005
```

## 四、版本号规则

### 4.1 格式

```
主版本号.次版本号.修订号
<major>.<minor>.<patch>

示例：2.4.0 → 2.4.1 → 2.4.2
```

### 4.2 更新规则

| 场景 | 更新规则 |
|------|----------|
| 合并到main | 版本号 +0.0.1 |
| 次版本更新 | 修订号归零，次版本号+1 |
| 主版本更新 | 次版本号归零，主版本号+1 |

### 4.3 需要更新的文件

| 文件 | 位置 |
|------|------|
| `pom.xml` | `<version>` |
| `application.yml` | `app.version` |

## 五、API设计规则

### 5.1 RESTful规范

| 方法 | 用途 | 示例 |
|------|------|------|
| GET | 查询 | `GET /api/requirements` |
| POST | 创建 | `POST /api/requirements` |
| PUT | 更新 | `PUT /api/requirements/{id}` |
| DELETE | 删除 | `DELETE /api/requirements/{id}` |

### 5.2 命名规范

- 资源名称使用复数形式
- 使用 kebab-case
- 嵌套资源使用路径参数

```bash
# 正确
GET /api/requirement-items
GET /api/requirements/{id}/comments

# 错误
GET /api/requirementItem
GET /api/getRequirements
```

## 六、数据库规则

### 6.1 表命名

- 使用下划线分隔
- 使用复数形式

```sql
-- 正确
CREATE TABLE requirements (...);
CREATE TABLE task_items (...);

-- 错误
CREATE TABLE Requirement (...);
CREATE TABLE taskItem (...);
```

### 6.2 字段命名

- 使用下划线分隔
- 使用小写

```sql
-- 正确
user_name, created_at, updated_at

-- 错误
userName, createdAt, CreatedAt
```

## 七、代码规范

### 7.1 Java规范

| 规范 | 要求 |
|------|------|
| 类名 | UpperCamelCase |
| 方法名 | lowerCamelCase |
| 常量 | UPPER_SNAKE_CASE |
| 包名 | 小写，用点分隔 |

### 7.2 前端规范

| 规范 | 要求 |
|------|------|
| 组件名 | PascalCase |
| 方法名 | camelCase |
| 常量 | UPPER_SNAKE_CASE |
| CSS类名 | kebab-case |

## 八、强制规则

### 8.1 Git操作

- ⚠️ 没有用户允许，禁止上传代码到远程分支
- ⚠️ 没有用户指令，禁止合并到main
- ⚠️ 合并到main时，版本号必须+0.0.1

### 8.2 工作流

- ⚠️ 每次用户对话必须调用工作流技能
- ⚠️ 所有评审由AI全员参与
- ⚠️ 全体通过才能进入下一阶段

### 8.3 知识沉淀

- ⚠️ 完成任务必须沉淀知识
- ⚠️ 发现问题必须记录
- ⚠️ 禁止知识流失
