## 1. 版本号管理

### 1.1 版本号格式

```
主版本号.次版本号.修订号
v<major>.<minor>.<patch>

示例：
v2.4.0  - 首次发布
v2.4.1  - 修订号增加（Bug修复）
v2.5.0  - 次版本号增加（新功能）
v3.0.0  - 主版本号增加（重大架构变更）
```

### 1.2 版本号更新规则

**每次代码提交必须更新版本号！**

需要更新的文件：

| 文件路径 | 更新位置 | 示例 |
|---------|---------|------|
| `hgsb-java/pom.xml` | `<version>` | `<version>2.4.6</version>` |
| `hgsb-java/src/main/resources/application.yml` | `app.version` | `app: version: 2.4.6` |
| `README.md` | 版本历史 | `### v2.4.6 (2026-03-26)` |

### 1.3 版本历史记录

在 `README.md` 的版本历史部分添加记录：

```markdown
## 版本历史

### v2.4.6 (2026-03-26)

- 添加mermaid流程图支持
- 修复目录导航跳转问题
- 优化移动端菜单

### v2.4.5 (2026-03-25)
- [更早的记录]
```

## 2. 代码提交规范

### 2.1 提交前检查

- [ ] 代码已通过 lint 检查
- [ ] 代码已通过测试
- [ ] 版本号已更新
- [ ] 提交信息符合规范
- [ ] 没有遗留调试代码

### 2.2 提交信息格式

```
<类型>(<范围>): <简短描述>

[可选的详细描述]

[可选的关联需求]
```

### 2.3 提交示例

```
# 好例子
feat(wms-manual): 添加mermaid流程图支持

- 集成mermaid库渲染18个流程图
- 修复{#section}显示问题
- 优化目录导航样式

关联需求: PRD-2026-002

# 坏例子
fix bug - 更新了

# 坏例子
WMS操作手册修改
```

### 2.4 类型说明

| 类型 | 说明 | 使用场景 |
|------|------|---------|
| `feat` | 新功能 | 添加新页面、新功能模块 |
| `fix` | Bug修复 | 修复功能缺陷 |
| `docs` | 文档 | README、规范文档等 |
| `style` | 格式 | 代码格式化、空格等 |
| `refactor` | 重构 | 代码优化但不改变功能 |
| `perf` | 性能 | 性能优化相关 |
| `test` | 测试 | 添加/修改测试 |
| `chore` | 工具 | 构建脚本、依赖更新 |

## 3. 前端开发规范

### 3.1 Vue组件规范

**组件命名**：
- 使用 PascalCase 命名
- 文件名与组件名一致
- 示例：`WMSManualView.vue`、`ContainerManagement.vue`

**组件结构**：
```vue
<template>
  <!-- 模板 -->
</template>

<script setup>
// 逻辑
</script>

<style scoped>
/* 样式 */
</style>
```

### 3.2 样式规范

- 使用 scoped 防止样式污染
- 优先使用 CSS 变量
- 遵循 BEM 命名规范（可选）
- 移动端响应式设计

### 3.3 API调用规范

```javascript
// 使用统一的API模块
import api from '@/api'

// 统一错误处理
try {
  const data = await api.getRecords()
} catch (error) {
  console.error('获取数据失败:', error)
}
```

## 4. 后端开发规范

### 4.1 Java代码规范

- 遵循阿里巴巴Java开发规范
- 类名使用 PascalCase
- 方法名、变量名使用 camelCase
- 常量使用 UPPER_SNAKE_CASE

### 4.2 分层结构

```
controller/     # 控制层，处理请求
service/        # 服务层，业务逻辑
repository/     # 数据访问层
entity/         # 实体类
dto/            # 数据传输对象
```

### 4.3 命名规范

```java
// Controller
@RestController
public class RecordsController { }

// Service
public interface RecordService { }
public class RecordServiceImpl implements RecordService { }

// Repository
public interface RecordRepository extends JpaRepository<Record, Long> { }

// Entity
@Entity
public class Record { }

// DTO
public class RecordDTO { }
```

## 5. 数据库规范

### 5.1 表命名

- 使用下划线分隔
- 使用单数名词
- 包含业务前缀（如适用）

```sql
CREATE TABLE operation_log (
    id BIGINT PRIMARY KEY,
    action_type VARCHAR(50),
    created_at TIMESTAMP
);
```

### 5.2 字段命名

- 使用下划线分隔
- 布尔字段使用 is_ 前缀
- 时间字段使用 _at 后缀

```sql
is_active BOOLEAN,
created_at TIMESTAMP,
updated_at TIMESTAMP
```
