# 完整AI开发工作流说明

**文档版本**：v1.0
**创建日期**：2026-03-27
**作者**：技术总监

---

## 一、工作流概览

```
用户输入需求 → AI主Agent接收 → 任务分解 → SubAgent执行 → 自动汇报 → 等待用户确认关键节点
```

---

## 二、角色定义

### 2.1 AI角色

| 角色 | 标识 | 职责 | 权限 |
|------|------|------|------|
| 主Agent | 🤖 | 总指挥、任务分解、进度监控 | 任务分配、进度汇报 |
| 需求Agent | 📋 | 需求分析、需求建模 | 创建/更新需求 |
| UI设计Agent | 🎨 | 界面设计、交互设计 | 添加设计说明 |
| 开发Agent | 💻 | 架构设计、代码实现 | 代码开发、创建Bug |
| 质量Agent | 🔍 | 测试用例、测试验证 | 创建/验证Bug |
| 部署Agent | 🚀 | 环境部署、版本发布 | 更新版本状态 |
| 知识管理Agent | 📚 | 文档整理、经验总结 | 记录项目总结 |

### 2.2 人类角色

| 角色 | 职责 | 必须参与环节 |
|------|------|-------------|
| **用户（您）** | 最终决策者 | 合并main、部署生产、需求变更确认 |

## 三、工作流阶段

### 阶段1：需求输入

| 项目 | 说明 |
|------|------|
| **输入** | 用户描述需求 |
| **执行者** | 主Agent |
| **输出** | 需求分析文档 |
| **自动/手动** | 🤖 自动 |
| **用户参与** | ❌ 不需要 |

**主Agent行为**：
1. 接收用户需求
2. 分析需求类型和优先级
3. 决定是否需要创建正式需求

---

### 阶段2：需求创建

| 项目 | 说明 |
|------|------|
| **输入** | 用户需求 |
| **执行者** | 需求Agent |
| **输出** | 需求文档（PRD-XXXX） |
| **自动/手动** | 🤖 自动 |
| **用户参与** | ❌ 不需要 |

**需求Agent行为**：
1. 在需求管理系统中创建需求
2. 填写需求描述、优先级、关联版本
3. 设置需求状态为"待评审"

---

### 阶段3：AI全员评审

| 项目 | 说明 |
|------|------|
| **输入** | 需求文档 |
| **执行者** | 所有AI Agent |
| **输出** | 评审报告 |
| **自动/手动** | 🤖 自动 |
| **用户参与** | ❌ 不需要 |

**评审参与**：
| Agent | 评审视角 | 评审输出 |
|-------|----------|----------|
| 需求Agent | 需求完整性 | 需求描述是否清晰 |
| UI设计Agent | 设计可行性 | UI是否可实现 |
| 开发Agent | 技术可行性 | 技术方案是否合理 |
| 质量Agent | 测试可行性 | 是否可测试 |
| 部署Agent | 部署可行性 | 部署复杂度 |
| 主Agent | 综合评估 | 最终结论 |

**通过标准**：所有Agent都输出"通过"

**不通过处理**：返回需求Agent修改 → 重新评审

---

### 阶段4：UI设计

| 项目 | 说明 |
|------|------|
| **输入** | 评审通过的需求 |
| **执行者** | UI设计Agent |
| **输出** | UI设计方案 |
| **自动/手动** | 🤖 自动 |
| **用户参与** | ❌ 不需要 |

**UI设计Agent行为**：
1. 设计页面布局
2. 定义交互流程
3. 在需求管理系统中添加设计说明

---

### 阶段5：UI评审

| 项目 | 说明 |
|------|------|
| **输入** | UI设计方案 |
| **执行者** | 所有AI Agent |
| **输出** | UI评审报告 |
| **自动/手动** | 🤖 自动 |
| **用户参与** | ❌ 不需要 |

---

### 阶段6：开发设计

| 项目 | 说明 |
|------|------|
| **输入** | UI设计 + 需求文档 |
| **执行者** | 开发Agent |
| **输出** | 技术设计方案、代码 |
| **自动/手动** | 🤖 自动 |
| **用户参与** | ❌ 不需要 |

**开发Agent行为**：
1. 技术架构设计
2. 数据库设计
3. API设计
4. 代码实现
5. 本地测试

---

### 阶段7：开发评审

| 项目 | 说明 |
|------|------|
| **输入** | 技术方案 + 代码 |
| **执行者** | 所有AI Agent |
| **输出** | 开发评审报告 |
| **自动/手动** | 🤖 自动 |
| **用户参与** | ❌ 不需要 |

---

### 阶段8：代码提交

| 项目 | 说明 |
|------|------|
| **输入** | 开发完成的代码 |
| **执行者** | 开发Agent |
| **输出** | Git提交记录 |
| **自动/手动** | 🤖 自动 |
| **用户参与** | ❌ 不需要 |

**Git操作流程**：
```
开发Agent → 创建sub分支 → 开发 → 提交到sub分支 → 合并到test分支
```

**分支命名**：`sub/<Agent类型>/<任务描述>`
**示例**：`sub/dev/add-task-management`

---

### 阶段9：测试

| 项目 | 说明 |
|------|------|
| **输入** | test分支代码 |
| **执行者** | 质量Agent |
| **输出** | 测试报告、Bug记录 |
| **自动/手动** | 🤖 自动 |
| **用户参与** | ❌ 不需要 |

**质量Agent行为**：
1. 设计测试用例
2. 执行功能测试
3. 记录Bug
4. 验证Bug修复

---

### 阶段10：部署

| 项目 | 说明 |
|------|------|
| **输入** | 测试通过的代码 |
| **执行者** | 部署Agent |
| **输出** | 部署完成 |
| **自动/手动** | 🤖 自动 |
| **用户参与** | ❌ 不需要 |

**部署Agent行为**：
1. 构建应用
2. 部署到测试环境
3. 验证部署结果

## 四、必须用户参与环节

以下环节**必须**由用户（您）确认：

| 环节 | 原因 | 用户操作 |
|------|------|----------|
| **合并到main** | 生产环境代码变更 | 发送指令："合并到main" |
| **部署生产** | 正式环境变更 | 发送指令："部署生产" |
| **需求重大变更** | 范围调整 | 确认新的需求范围 |

**用户只需发送简单指令**：
- "合并到main"
- "部署生产"
- "需求变更确认"

以下环节**完全自动**执行，无需用户参与：

| ✅ UI设计 | UI设计Agent自动设计 |

| ✅ 代码提交 | 自动Git操作 |
| ✅ 测试执行 | 质量Agent自动测试 |
| ✅ 部署测试环境 | 部署Agent自动部署 |
| ✅ 状态更新 | 在需求管理系统中更新 |
